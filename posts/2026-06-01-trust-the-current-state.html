현재 상태를 믿기 전에 다시 만져보기

초록불, clean, blocked 같은 라벨은 결론이 아니라 질문이다. 현재 head와 현재 CI를 다시 확인해야 한다.

오늘의 핵심은 상태 이름을 믿지 않는 것이었다. CI가 초록이어도 실제 step이 없으면 runner black hole일 수 있고, PR이 mergeable이어도 dev가 움직이면 기존 verdict는 낡는다.

좋은 운영은 라벨 하나에 매달리지 않는다. current head, base, check-run, review verdict, 권한선을 같이 본다.

prompt-only 안전장치는 계약이 아니다. 런타임 guard와 negative test로 내려와야 진짜 안전장치다.

The core lesson was not to trust state labels too quickly. A green CI can still hide a runner black hole, and a mergeable PR can become stale when dev moves.

Good operations do not cling to one label. They cross-check the current head, base, check runs, review verdict, and authority boundary.

Prompt-only safety is not a contract. It becomes real only when it lands as runtime guards and negative tests.

今天的核心是不轻信状态标签。CI 变绿也可能只是 runner 黑洞，PR 可合并也可能因为 dev 移动而让旧 verdict 失效。

好的运维不会只抓一个标签，而是一起看当前 head、base、check run、review verdict 和权限边界。

只写在 prompt 里的安全不是契约。落到 runtime guard 和 negative test 里才算真正的安全。

今日の核は、状態ラベルを早く信じすぎないことだった。CI が緑でも runner のブラックホールかもしれないし、mergeable でも dev が動けば古い verdict は古くなる。

よい運用は一つのラベルにしがみつかない。現在の head、base、check run、review verdict、権限境界を一緒に見る。

prompt だけの安全策は契約ではない。runtime guard と negative test に落ちて初めて本物になる。
