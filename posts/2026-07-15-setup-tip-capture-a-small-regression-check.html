수정 내용을 작은 검사로 고정하세요

버그를 고친 뒤 전체 테스트만 통과시키고 끝내면 어떤 경계가 실제로 보호되는지 흐려질 수 있습니다. 수정 전에는 실패하고 수정 후에는 통과하는 가장 작은 입력을 별도 검사로 남기세요.

검사는 정상 경로만 반복하지 말고 문제가 발생한 경계값, 오류 입력, 상태 전환을 직접 다뤄야 합니다. 이름에는 기대 동작과 재발하면 안 되는 실패를 드러내는 편이 좋습니다.

가능하면 빠르고 결정적으로 실행되도록 외부 네트워크나 시간 의존성을 제거하세요. 작은 회귀 검사는 로컬 반복 실행과 CI 모두에서 부담 없이 돌아가야 합니다.

마지막으로 실제 생성물이나 사용자 경로가 관련된 버그라면 해당 표면도 한 번 확인하세요. 단위 검사는 원인을 고정하고, 짧은 스모크 검사는 통합 결과를 확인합니다.

Pin the fix with a small check

If a bug fix ends with only a full test-suite pass, the protected boundary can remain unclear. Save the smallest input that failed before the fix and passes afterward as a focused regression check.

Do not repeat only the happy path. Exercise the boundary value, invalid input, or state transition that exposed the defect. Name the check so both the expected behavior and the forbidden regression are obvious.

Keep it fast and deterministic by removing unnecessary network and clock dependencies. A small regression check should be cheap enough to run repeatedly on a workstation and in CI.

When the bug affected generated output or a user-facing path, smoke-test that surface too. The focused test pins the cause; the short smoke check confirms the integrated result.

用一个小检查固定修复成果

如果修复缺陷后只运行完整测试套件，真正受到保护的边界仍可能不清楚。应保存一个最小输入：修复前会失败，修复后能够通过。

不要只重复正常路径。直接覆盖暴露问题的边界值、无效输入或状态转换，并让检查名称清楚表达预期行为与禁止再次出现的故障。

去掉不必要的网络和时间依赖，使检查保持快速且确定。小型回归检查应当足够轻量，能在本地和 CI 中频繁运行。

若缺陷影响了生成产物或用户路径，还应对该表面做一次简短冒烟测试。聚焦测试锁定原因，冒烟检查确认集成结果。

小さなチェックで修正を固定する

バグ修正後にテスト全体を通すだけでは、どの境界が守られたのか曖昧になりがちです。修正前は失敗し、修正後は通る最小の入力を回帰チェックとして残します。

正常系だけを繰り返さず、問題を露出させた境界値、不正入力、状態遷移を直接扱います。名前から期待動作と再発してはいけない失敗が分かるようにします。

不要なネットワークや時刻依存を除き、高速で決定的に実行できるようにします。小さな回帰チェックはローカルでも CI でも気軽に繰り返せるべきです。

生成物や利用者向け経路に影響した不具合なら、その表面も短く確認します。焦点を絞ったテストが原因を固定し、スモークチェックが統合結果を確かめます。
