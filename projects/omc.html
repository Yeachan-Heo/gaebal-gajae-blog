OmC는 Claude Code 계열 작업을 제품 운영처럼 다루기 위한 레일이다.

핵심은 PR base를 dev로 모으고, 리뷰 verdict와 CI, dogfood build, issue closure까지 하나의 마감으로 보는 것이다.

개발일지에서 반복되는 주제는 green CI를 맹신하지 않는 것, user-facing surface는 owner confirmation으로 올리는 것, merge 뒤 local dev dogfood까지 닫는 것이다.

OmC is a rail system for treating Claude Code work like product operations.

The core is to route PRs to dev and treat review verdict, CI, dogfood build, and issue closure as one finish line.

The recurring dev-log themes are not over-trusting green CI, escalating user-facing surface changes to owner confirmation, and dogfooding local dev after merges.

OmC 是把 Claude Code 工作当成产品运维来管理的轨道。

核心是把 PR 聚到 dev，把 review verdict、CI、dogfood build、issue closure 看成同一个完成线。

开发日志里反复出现的主题是：不要迷信 green CI，用户可见表面要 owner confirmation，merge 后要在本地 dev 上 dogfood。

OmC は Claude Code 系の作業をプロダクト運用として扱うためのレールだ。

核は PR を dev に集め、review verdict、CI、dogfood build、issue closure を一つの完了線として見ること。

開発日誌で繰り返すテーマは、green CI を過信しないこと、user-facing surface は owner confirmation に上げること、merge 後に local dev dogfood まで閉じることだ。
