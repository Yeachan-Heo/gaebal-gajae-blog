# 가재 블로그 디자인 시스템 파일럿 — 프로젝트 브리프

## 이 PR의 역할
이 작업은 기존 화면을 조금 더 세련되게 다듬는 UI polish가 아니다.
목적은 **가재 블로그 전용의 새 디자인 시스템**을 먼저 정의하고, 이후 화면 작업이 그 시스템을 반복적으로 따르도록 뼈대를 만드는 것이다.

## 이번에 바로잡는 오해
- 기존 화면의 톤을 유지한 채 살짝 예쁘게 고치는 것이 아니다.
- 기존 다크 셸/카드/라운드 박스 언어를 답습하는 것도 아니다.
- 색상만 바꾸는 리스킨도 아니다.

이번 파일럿은 **색상, 표면, 서체 위계, 컴포넌트 실루엣, 증거 블록 언어**까지 새로 잡는다.

## 제품 맥락
가재 블로그는 마케팅 랜딩이 아니라,
공개 가능한 판단·교정·셋업·사고 기록을 남기는 **기술 운영 로그북**이다.

사용자가 느껴야 하는 것은 다음에 가깝다.
- 따뜻한 기술 현장 노트
- 개발 에이전트 운영 일지
- 터미널 로그와 검증 영수증
- 사고 보고서/스탬프/배지

## 디자인 배합
- Warm Field Notebook 60%
- Terminal Proof Lab 30%
- Playful Incident Report 10%

## 성공 기준
1. `docs/style-foundry/` 아래에 새 디자인 시스템 문서가 정리되어 있다.
2. `assets/styles/00-tokens.css`가 새 팔레트와 스케일의 기준점이 된다.
3. `assets/styles/20-components.css`가 공용 컴포넌트와 가재 전용 컴포넌트를 분리한다.
4. `style-foundry-preview.html`에서 토큰과 컴포넌트를 한 화면에서 점검할 수 있다.
5. 홈/아카이브/포스트/프로젝트 화면이 즉흥 스타일이 아니라 새 토큰과 컴포넌트를 따른다.

## 비목표
- 이번 PR에서 최종 브랜드 완성형을 확정하는 것
- 애니메이션/마케팅 연출을 넣는 것
- 범용 SaaS 템플릿처럼 보이는 것

## 작업 원칙
앞으로 프론트 작업은 화면부터 그리지 않는다.
먼저 토큰을 고르고, 그다음 컴포넌트를 고르고, 마지막에 페이지를 조립한다.
